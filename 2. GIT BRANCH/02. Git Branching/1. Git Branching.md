### ğŸ¯ Kenapa Perlu Membuat Branch Baru?

Saat kamu ingin menambah fitur baru atau memperbaiki bug, **sebaiknya jangan langsung dikerjakan di branch utama (master)**. Kenapa? Karena kalau ada kesalahan, branch utama tidak akan terganggu!

### ğŸ“ Apa yang Terjadi Saat Membuat Branch Baru?

Ketika kamu membuat branch baru, Git akan **menyalin semua isi** dari branch tempat kamu berada saat itu. Jadi branch baru akan memiliki history yang sama sampai titik tersebut.

### ğŸ› ï¸ Cara Membuat Branch Baru

- Untuk membuat branch baru, caranya sangat mudah, kita bisa gunakan perintah:

```bash

```

**Contoh:**
```bash

```

> âš ï¸ **Penting:** Saat kita buat branch baru, secara otomatis branch baru akan dimulai dari posisi branch kita saat ini. Jika misal kita membuat branch baru ketika posisi kita sedang di branch master, maka branch baru tersebut akan dibuat dari branch master.

## ğŸ”„ Pindah ke Branch Lain

### ğŸš¶ Kenapa Perlu Pindah Branch?

Setelah membuat branch baru, kamu **tidak otomatis pindah** ke branch tersebut. Kamu masih berada di branch lama! Untuk mulai bekerja di branch baru, kamu harus **pindah** dulu.

### ğŸ¯ Kapan Harus Pindah Branch?

- Saat ingin mengerjakan fitur di branch yang berbeda
- Saat ingin memeriksa kode di branch lain
- Saat ingin melanjutkan pekerjaan yang tertunda di branch tertentu

### ğŸš€ Cara Pindah Branch

- Untuk pindah ke branch lain, kita perlu lakukan secara manual
- Caranya kita bisa gunakan perintah:

**Cara Modern (Disarankan):**
```bash

```

**Cara Klasik:**
```bash

```

**Contoh:**
```bash

```

**Output:**
```

```

> ğŸ†• **Rekomendasi:** Gunakan `git switch` karena lebih modern dan khusus untuk berpindah branch. Perintah `git checkout` lebih multifungsi tapi bisa membingungkan pemula.

### ğŸ¤¦ Kenapa Perlu Mengubah Nama Branch?

### ğŸ“Œ Langkah-Langkah Mengubah Nama

### ğŸ”§ Cara Mengubah Nama Branch

- Jika kita melakukan kesalahan pembuatan nama branch, kita juga bisa lakukan perubahan nama branch

- Setelah pindah, kita bisa gunakan perintah:

> ğŸ“ **Catatan:** Flag `-m` adalah singkatan dari **move/rename**. Nama branch akan langsung berubah!

## ğŸ—‘ï¸ Menghapus Branch

### ğŸ§¹ Kenapa Perlu Menghapus Branch?

Branch yang sudah tidak dipakai akan **menumpuk** dan membuat repository berantakan. Bayangkan kamu punya 50 branch lama yang tidak terpakai - susah mencari yang masih aktif!

### âš ï¸ Hal yang Perlu Diperhatikan

- âŒ **Tidak bisa menghapus branch yang sedang aktif** - kamu harus keluar dulu
- âœ… **Pastikan pekerjaanmu sudah di-merge** ke branch utama sebelum dihapus
- ğŸ’¡ Branch yang sudah dihapus **tidak bisa dikembalikan** (kecuali dengan cara advanced)

### ğŸ”¥ Cara Menghapus Branch

- Jika sebuah branch sudah tidak digunakan lagi, idealnya kita perlu menghapus branch tersebut
- Untuk menghapus branch, kita perlu keluar terlebih dahulu dari branch tersebut
- Lalu gunakan perintah:

```bash
git branch -d namabranch
```

```bash
git branch --delete namabranch
```

**Contoh lengkap:**
```bash
# 1. Pastikan tidak berada di branch yang akan dihapus
git switch master

# 2. Hapus branch
git branch -d feature-login
```

**Output jika berhasil:**
```
Deleted branch feature-login (was 2313abc).
```

**Output jika branch belum di-merge:**
```
error: The branch 'feature-login' is not fully merged.
```

> âš¡ **Tips:** Jika kamu yakin ingin menghapus branch meskipun belum di-merge, gunakan `-D` (huruf besar) sebagai force delete: `git branch -D namabranch`
