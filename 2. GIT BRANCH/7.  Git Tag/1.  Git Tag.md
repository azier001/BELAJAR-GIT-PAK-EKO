## ğŸ“– Apa itu Tag?

**Tag** adalah fitur di Git yang memungkinkan kita untuk **menandai** sebuah commit tertentu dengan nama yang mudah diingat.

### ğŸ¤” Mengapa Tag Penting?

Bayangkan kamu sudah membuat ratusan commit di project kamu. Bagaimana cara kamu mengingat commit mana yang merupakan versi 1.0.0 dari aplikasi kamu? Commit ID seperti `a7b3c9d` sangat sulit diingat bukan? 

Nah, di sinilah **Tag** berperan! ğŸ¯

### âœ¨ Keunggulan Tag

- **Tag merupakan fitur** yang memungkinkan kita menandai sebuah commit id dengan nama yang bermakna
- **Sebelumnya kita sudah tahu dengan HEAD**, yaitu reference (penunjuk) ke commit terakhir di branch kita saat ini
- **Jika kita ingin membuat sebuah reference ke sebuah commit**, kita bisa menggunakan Tag
- **Dalam pengembangan perangkat lunak**, biasanya Tag digunakan sebagai **penanda versi rilis** dari aplikasi, misal Tag `1.0.0`, Tag `1.0.2`, dan lain-lain
- **Karena Tag merupakan reference ke commit**, jadi Tag bisa dilakukan di branch manapun

> ğŸ’¡ **Tip**: Tag seperti bookmark untuk commit penting di Git kamu!

## ğŸ¯ Membuat Tag

### ğŸ“ Aturan Penting

**Tag adalah sesuatu yang unik!** âš ï¸

Artinya, jika kita sudah membuat tag dengan nama `v1.0.0`, maka kita **tidak bisa** membuat tag dengan nama yang sama lagi. Setiap tag harus memiliki nama yang berbeda.

### ğŸ› ï¸ Cara Membuat Tag

Untuk membuat tag, kita bisa gunakan perintah:

```bash
git tag tagName commitId
```

**Penjelasan:**
- `tagName` = Nama tag yang ingin kamu buat (contoh: `v1.0.0`, `release-1.0`)
- `commitId` = ID commit yang ingin kamu tandai (contoh: `a7b3c9d`)

**Contoh Penggunaan:**
```bash
git tag v1.0.0 a7b3c9d
```

> ğŸ’¡ **Tip**: Gunakan format penamaan yang konsisten seperti `v1.0.0`, `v1.0.1`, dll untuk memudahkan tracking versi!

## âª Checkout ke Tag

**Checkout** adalah cara kita untuk "berpindah" ke kondisi project di masa lalu. Seperti mesin waktu! â°

### ğŸ¤” Kapan Kita Perlu Checkout ke Tag?

- Ketika ingin melihat kode pada versi tertentu
- Ketika ingin membandingkan versi lama dengan versi baru
- Ketika ingin membuat branch dari versi tertentu
- Ketika perlu rollback ke versi yang stabil

### ğŸ› ï¸ Cara Checkout ke Tag

**Seperti yang sudah dijelaskan di kelas Git Dasar**, kita bisa melihat snapshot (kondisi) sebelumnya di Git.

**Sekarang dengan menggunakan Tag**, kita juga bisa melihat snapshot sebelumnya tanpa harus menggunakan commitId yang panjang dan sulit diingat!

Cukup menggunakan nama Tag:

```bash
git checkout tagname
```

**Contoh Penggunaan:**
```bash
git checkout v1.0.0
```

**Apa yang terjadi?**

Setelah menjalankan perintah di atas:
- Project kamu akan kembali ke kondisi saat versi `1.0.0`
- File-file akan berubah sesuai kondisi saat tag `v1.0.0` dibuat
- Kamu bisa melihat dan menjalankan kode pada versi tersebut

> âš ï¸ **Peringatan**: Saat checkout ke tag, kamu akan masuk ke mode "detached HEAD". Ini normal! Untuk kembali ke branch utama, gunakan `git checkout main` atau `git checkout master`

**Untuk kembali ke branch utama:**
```bash
git checkout main
```

### ğŸ”„ Cara Memindahkan Tag ke Commit Lain

