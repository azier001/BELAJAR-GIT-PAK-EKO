## ğŸ“š Apa Itu Compare Commit?

**Compare Commit** adalah fitur Git yang memungkinkan kita untuk **membandingkan snapshot (hasil akhir)** dari dua commit yang berbeda.

---

## ğŸ¯ Konsep Penting yang Harus Dipahami

### âœ… Yang Benar: Membandingkan Snapshot

- Git membandingkan **hasil akhir** dari setiap commit
- Bukan membandingkan **proses perubahan** yang terjadi di antara commit
- Yang dibandingkan adalah **kondisi file saat itu**, bukan histori perubahannya

### âŒ Kesalahan Umum

Banyak pemula mengira compare commit akan menampilkan **semua perubahan** yang pernah terjadi antara dua commit. Ini **SALAH**! 

---

## ğŸ’¡ Contoh Sederhana

Bayangkan kita punya tiga commit:

1. **Commit Pertama**: Ada `file1.txt` dan `file2.txt`
2. **Commit Kedua**: Menambahkan `file3.txt`
3. **Commit Ketiga (HEAD)**: Menghapus `file3.txt`

### ğŸ¤” Pertanyaan: Apa yang akan muncul jika kita bandingkan Commit Pertama dengan Commit Ketiga?

**Jawaban**: Hanya perbandingan antara `file1.txt` dan `file2.txt`

**Mengapa `file3.txt` tidak muncul?** 
- Karena pada Commit Pertama, `file3.txt` **belum ada**
- Pada Commit Ketiga, `file3.txt` **sudah tidak ada**
- Jadi Git hanya membandingkan file yang **ada di kedua commit**

## ğŸ› ï¸ Cara Menggunakan Compare Commit

### Perintah Dasar

```bash
git diff hash1 hash2
```

**Penjelasan:**
- `hash1` = Hash commit pertama (commit lama)
- `hash2` = Hash commit kedua (commit baru)
- Perintah ini akan menampilkan perbedaan dalam format **diff**

### Perintah yang Digunakan

- `7453f5e` = Hash dari commit lama
- `HEAD` = Commit terakhir (saat ini)

### ğŸ” Membaca Output Diff

#### Untuk `file1.txt`:
- **Tanda `-` (merah)**: Baris yang **dihapus** atau **diubah**

- `new file mode 100644`: File **baru** yang dibuat
- `+Ini adalah file 2`: Isi dari file baru tersebut

## ğŸ–¥ï¸ Difftool - Melihat Perubahan dengan Visual Studio Code

### Apa Itu Difftool?

**Difftool** adalah cara untuk melihat perbedaan commit menggunakan **aplikasi eksternal** seperti Visual Studio Code, yang lebih **mudah dibaca** dan **user-friendly**.

### ğŸ“‹ Syarat Menggunakan Difftool

Sebelumnya kita harus sudah melakukan **pengaturan** untuk menggunakan Visual Studio Code sebagai difftool.

### Perintah Difftool

```bash
git difftool hash1 hash2
```

**Contoh:**
```bash
git difftool 7453f5e HEAD
```
