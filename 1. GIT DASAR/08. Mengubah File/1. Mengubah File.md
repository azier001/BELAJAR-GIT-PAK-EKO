Setelah kita membuat file di repository, tentu saja kita akan melakukan perubahan pada file tersebut. Git dirancang untuk mendeteksi setiap perubahan yang terjadi pada file di dalam repository.

## ğŸ” Melihat Perubahan File Secara Detail

### ğŸ¯ Apa itu Git Diff?

Ketika kita melakukan perubahan pada file, Git tidak hanya mendeteksi **bahwa file berubah**, tetapi juga bisa menunjukkan **apa yang berubah** secara detail.

Untuk melihat perubahan yang terjadi, gunakan perintah:

```bash

```

-Ini adalah file 3

### ğŸ“– Penjelasan Output Git Diff

Mari kita pahami output di atas baris per baris:

| Baris | Penjelasan |
|-------|------------|
| `diff --git a/file3.txt b/file3.txt` | Membandingkan versi **a** (lama) dengan versi **b** (baru) dari file3.txt |
| `index a2a9337..aaf8c5a 100644` | Informasi hash (ID unik) dari kedua versi file |
| `--- a/file3.txt` | Tanda untuk versi **lama** file |
| `+++ b/file3.txt` | Tanda untuk versi **baru** file |
| `@@ -1 +1,2 @@` | Perubahan terjadi: dari baris 1 menjadi baris 1-2 |
| `-Ini adalah file 3` | Baris yang **dihapus** (ditandai dengan `-` dan warna merah) |
| `+Ini adalah file 3` | Baris yang **ditambahkan** (ditandai dengan `+` dan warna hijau) |
| `+Menambah baris di file3.txt` | Baris baru yang **ditambahkan** |

### ğŸ¨ Memahami Warna pada Git Diff

- ğŸ”´ **Merah dengan tanda `-`** = Konten yang **dihapus** atau **diubah** dari versi lama
- ğŸŸ¢ **Hijau dengan tanda `+`** = Konten yang **ditambahkan** atau **versi baru**
- âšª **Putih** = Konten yang **tidak berubah** (konteks)
