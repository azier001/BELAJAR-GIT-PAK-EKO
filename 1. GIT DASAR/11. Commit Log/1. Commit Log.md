## ğŸ¯ Apa itu Commit Log?

Git adalah **distributed version control**, artinya walaupun Repository ada di komputer lokal kita, semua riwayat perubahan tetap tersimpan di komputer kita sendiri.

### âš–ï¸ Kelebihan dan Kekurangan

**Kekurangan:**
- Repository akan semakin besar ukurannya seiring waktu karena menyimpan semua riwayat

**Kelebihan:**
- Kita bisa melihat semua riwayat commit kapan saja
- Tidak perlu koneksi internet untuk melihat history
- Riwayat ini disebut **Commit Log**

## ğŸ¨ Log Sederhana: `git log --oneline`

Kadang kita hanya ingin melihat **commit log message** saja atau istilahnya adalah versi sederhananya saja.

Untuk melakukan itu, kita bisa gunakan perintah:

```bash
git log --oneline
```

### ğŸ’¡ Penjelasan:
Perintah ini menampilkan log dalam **format ringkas**, hanya menampilkan:
- Hash commit yang dipendekkan (7 karakter pertama)
- Pesan commit saja

Sangat berguna ketika kita ingin **melihat ringkasan** tanpa detail yang terlalu banyak!

### ğŸ“‹ Contoh Output:

```

2118ac3 perubahan di file3.txt

4f15c9b menambah file2.txt

(END)
```

- Memahami alur development secara visual

## ğŸ” Melihat Detail Commit: `git show`

Kadang kita ingin melihat **detail perubahan** yang terjadi pada sebuah commit.

Untuk melakukan itu, kita bisa gunakan perintah:

```bash
git show hash
```

### ğŸ’¡ Penjelasan:
Perintah ini menampilkan **informasi lengkap** dari sebuah commit, termasuk:
- Informasi commit (author, date, message)
- **Diff lengkap** yang menunjukkan perubahan pada setiap file
- Baris yang ditambah (+) dan dihapus (-)

**Catatan:** Ganti `hash` dengan hash commit yang ingin dilihat (bisa menggunakan hash pendek dari `git log --oneline`)

### ğŸ“‹ Contoh Output:

```
commit 1b5f5645b4f13233ce228b57db7babe4e4b7ef61
Author: John Doe <john.doe@example.com>
Date:   Sat Jun 19 09:11:13 2021 +0700

    perubahan di file1.txt dan file2.txt

diff --git a/file1.txt b/file1.txt
index c7a8c45..f61cc80 100644
--- a/file1.txt
+++ b/file1.txt
@@ -1 +1,2 @@
-Ini adalah file 1
\ No newline at end of file
+Ini adalah file 1
+Menambah baris di file1.txt
\ No newline at end of file
diff --git a/file2.txt b/file2.txt
index 756daa2..8d28041 100644
--- a/file2.txt
+++ b/file2.txt
@@ -1 +1,2 @@
-Ini adalah file 2
\ No newline at end of file
+Ini adalah file 2
+Menambah baris di file2.txt
\ No newline at end of file
(END)
```

**Membaca Diff:**
- `---` = File sebelum perubahan
- `+++` = File setelah perubahan
- `-` (merah) = Baris yang dihapus
- `+` (hijau) = Baris yang ditambah
- `@@` = Informasi lokasi perubahan

## ğŸ’ª Tips untuk Pemula

1. **Mulai dengan `git log --oneline`** - Lebih mudah dibaca untuk pemula
2. **Gunakan `q` untuk keluar** - Tekan `q` saat melihat log untuk kembali ke terminal
3. **Copy hash yang pendek saja** - Cukup 7 karakter pertama untuk `git show`
4. **Praktik membaca diff** - Sangat berguna untuk tracking perubahan code
