### 1. ğŸš¨ Hati-hati dengan Merge Conflict

Ketika melakukan pull, kadang terjadi **merge conflict**. Ini terjadi ketika:
- Kita mengubah file yang sama dengan rekan kita
- Di baris kode yang sama
- Dan keduanya melakukan perubahan berbeda

**Solusinya:** Perbaiki secara manual seperti yang sudah dipelajari di materi merge.

### 2. âœ… Pull Otomatis Melakukan Fetch

Saat kita melakukan pull, Git secara otomatis akan:
1. Fetch terlebih dahulu (mengambil perubahan)
2. Kemudian merge (menggabungkan perubahan)

Jadi kita tidak perlu melakukan fetch secara terpisah!
