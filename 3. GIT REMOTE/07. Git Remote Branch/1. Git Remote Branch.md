- Branch dengan tanda `*` = branch yang sedang aktif
- Branch tanpa prefix `remotes/origin/` = branch lokal

### ğŸ¯ Kenapa Perlu Membuat Branch dari Remote?

Karena secara default hanya branch utama yang dibuat di lokal saat kita `clone`, kadang kita perlu bekerja dengan branch lain yang ada di remote. Misalnya:

- Branch `development` untuk pengembangan fitur
- Branch `feature-login` untuk fitur login
- Branch `bugfix-payment` untuk memperbaiki bug

Untuk bisa bekerja dengan branch tersebut, kita harus membuatnya di lokal terlebih dahulu.

---

### ğŸ“ Cara Membuat Local Branch dari Remote Branch

Gunakan perintah berikut:

```bash
git checkout -b localbranch namaremote/remotebranch
```

**Penjelasan Detail:**
- `git checkout` = perintah untuk berpindah atau membuat branch
- `-b` = singkatan dari **branch**, artinya buat branch baru
- `localbranch` = **nama branch lokal** yang ingin kita buat (bisa disesuaikan)
- `namaremote/remotebranch` = **nama remote dan branch** yang ingin kita salin (biasanya `origin/nama-branch`)

## ğŸš€ Tips untuk Pemula

1. **Selalu cek daftar branch** dengan `git branch -a` sebelum membuat branch baru

## â“ FAQ (Pertanyaan yang Sering Ditanyakan)

**Q: Apakah harus membuat branch lokal untuk setiap remote branch?**  
A: Tidak harus. Buat hanya branch yang memang akan kita gunakan untuk bekerja.

**Q: Bagaimana jika nama branch remote berubah?**  
A: Gunakan `git branch -r` untuk melihat nama terbaru, lalu buat branch lokal yang baru.

**Q: Apakah data di branch lokal otomatis sama dengan remote?**  
A: Ya, saat pertama dibuat akan sama. Tapi untuk update selanjutnya, gunakan `git pull`.
