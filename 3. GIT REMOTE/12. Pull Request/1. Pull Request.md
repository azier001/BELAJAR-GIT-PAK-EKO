### 1ï¸âƒ£ **Bekerja di Branch Terpisah**

Saat kita bekerja dengan tim, biasanya **kita tidak langsung mengubah branch utama** (main/master). Sebaliknya, kita membuat branch baru yang terpisah untuk pekerjaan kita.

> ðŸ’¡ **Mengapa begini?** Agar pekerjaan kita tidak mengganggu pekerjaan orang lain di branch utama.

### 2ï¸âƒ£ **Merge ke Branch Utama**

Setelah pekerjaan di branch kita selesai, kita bisa menggabungkan (merge) perubahan kita ke branch utama.

> ðŸ’¡ **Apa itu merge?** Proses menggabungkan kode dari branch satu ke branch lain.

```bash

# Output:
# Updating 1a2b3c4..5d6e7f8
# Fast-forward
#  login.js | 50 ++++++++++++++++++++
#  1 file changed, 50 insertions(+)
```

---

### 3ï¸âƒ£ **Pull Request vs Merge Manual**

Sebelum langsung merge, kita bisa menggunakan **Pull Request** untuk memberikan kesempatan pada rekan tim untuk **review kode kita** terlebih dahulu!

| Aspek | Merge Manual | Pull Request |
|-------|--------------|--------------|
| Proses | Langsung merge | Review dulu, baru merge |
| Keamanan | Rawan error | Lebih aman |
| Kolaborasi | Minimal | Maksimal |

> ðŸ” **Keuntungan Pull Request:** Tim bisa memberikan masukan sebelum kode digabung ke branch utama.

```bash
# Contoh workflow Pull Request:
# 1. Push branch kita ke remote
git push origin fitur-login

# Output:
# Enumerating objects: 5, done.
# Counting objects: 100% (5/5), done.
# Writing objects: 100% (5/5), 450 bytes
# To github.com:username/project.git
#  * [new branch]      fitur-login -> fitur-login

# 2. Buka GitHub dan buat Pull Request
# 3. Tunggu review dari tim
# 4. Merge setelah disetujui
```
