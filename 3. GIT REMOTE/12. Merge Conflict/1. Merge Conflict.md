## ğŸ“– Bagian 1: Pengenalan Merge Conflict

### ğŸ¤” Apa itu Merge Conflict?

Merge Conflict adalah **error atau peringatan** yang muncul saat kamu mencoba menggabungkan dua versi kode yang berbeda. Bayangkan seperti ini:

> **Analogi Sederhana:** 
> Kamu dan temanmu sama-sama mengubah baris yang sama di file yang sama, kemudian Git bingung versi mana yang harus dipakai. Inilah Merge Conflict! ğŸ¤·

### ğŸŒ³ Kapan Merge Conflict Terjadi?

Merge Conflict terjadi dalam dua situasi umum:

1. **Ketika melakukan Pull Request** 
   - Kadang sering terjadi Merge Conflict yang menyebabkan tidak bisa lakukan proses merge otomatis
   - Ini terjadi karena ada perubahan yang sama di dua branch berbeda

2. **Saat Merge dari Branch Asal**
   - Pada kasus seperti itu, kita perlu melakukan merge dari branch asal nya terlebih dahulu
   - Setelah itu, barulah kita bisa memperbaiki conflict nya

Ketika terjadi Merge Conflict, Git akan memberikan pesan seperti ini di terminal:

### ğŸ“„ Struktur Conflict di File

Git menandai bagian yang conflict dengan simbol khusus. Buka file tersebut dan cari ini:


**Penjelasan Simbol:**
